# Jorge Acevedo de León — CV / Portfolio Website

Static portfolio site ready for GitHub Pages.

## What is included

- `index.html`: main portfolio page.
- `styles.css`: responsive styling.
- `assets/Jorge_Acevedo_CV.pdf`: downloadable CV.
- `.nojekyll`: disables Jekyll processing for a plain static site.

## Recommended repository name

For a personal GitHub Pages site, create a repository named:

```txt
YOUR_GITHUB_USERNAME.github.io
```

Replace `YOUR_GITHUB_USERNAME` with your real GitHub username.

## Deploy through GitHub Pages

1. Create a new public GitHub repository.
2. Upload these files to the root of the repository:
   - `index.html`
   - `styles.css`
   - `.nojekyll`
   - the `assets/` folder
3. Go to the repository on GitHub.
4. Open **Settings → Pages**.
5. Under **Build and deployment**, select:
   - Source: **Deploy from a branch**
   - Branch: **main**
   - Folder: **/** root
6. Save.

Your site should become available at:

```txt
https://YOUR_GITHUB_USERNAME.github.io/
```

## Things to edit before publishing

Search for `YOUR_GITHUB_USERNAME` and replace it with your actual GitHub username.

Recommended edits:

- Add your GitHub profile URL.
- Add links to real repositories in the Projects section.
- Keep the phone number out of the public site unless you intentionally want recruiters to see it.
- If you buy a domain later, configure it in GitHub Pages and optionally add a `CNAME` file.
